# Безопасность

Токен и ID не коммитятся; `.env` исключён. ORM использует параметры вместо конкатенации SQL. Доступ к записи связывается с внутренним user ID, полученным из Telegram update. Админский Router фильтруется на каждом update. Callback разбираются типизированными `CallbackData`; идентификаторы преобразуются в UUID. Фото не скачиваются и ограничиваются конфигурацией. Структурированные логи не содержат токен и полный телефон. Для production ограничьте доступ к БД сетью Compose, ротируйте токен при утечке и установите retention персональных данных.
