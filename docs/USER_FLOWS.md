# Пользовательские сценарии

Во всех FSM-сценариях доступны «Назад», «Главное меню» и «Отменить». «Назад» сохраняет уже проверенные данные и возвращает к предыдущему состоянию; «Отменить» очищает FSM и временный hold, если он ещё принадлежит пользователю. Устаревший callback не меняет данные и показывает сообщение «Кнопка устарела».

## Первый запуск и навигация

- **Старт:** `/start`.
- **Шаги:** upsert User по `telegram_id`, обновление username/имени, приветствие и главное меню.
- **Кнопки:** «Записаться», «Услуги и цены», «Рассчитать стоимость», «Моя запись», «Частые вопросы», «Связаться с менеджером».
- **FSM:** отсутствует.
- **Ошибки:** при недоступной БД — нейтральное сообщение, событие в лог; дубликат пользователя невозможен из-за уникального Telegram ID.

## Запись: автомобиль и услуги

- **Старт:** «Записаться на бесплатный осмотр».
- **FSM:** `booking.brand → model → year → vehicle_class → services → summary`.
- **Шаги:** популярная марка/поиск/страницы/«Другой бренд»; фильтр моделей марки/ручной ввод; год 1980…текущий+1; класс; одна или несколько активных услуг из БД; итоговая карточка.
- **Кнопки:** марка/модель/страницы, «Другой бренд», «Ввести модель», услуги с toggle, «Продолжить», «Всё верно», «Изменить автомобиль», «Изменить услуги».
- **Переходы:** продолжение возможно только после валидного автомобиля и хотя бы одной услуги; изменения возвращают в нужное состояние без потери других данных.
- **Ошибки:** пустое/слишком длинное название, неверный год, неактивная или устаревшая услуга, продолжение без услуг.

## Фото

- **Старт:** подтверждённые автомобиль и услуги.
- **FSM:** `booking.photos`.
- **Шаги:** клиент отправляет 0…N фото, каждое сохраняется с `file_id`, `unique_file_id`, MIME, размером и порядком; после каждого отображается счётчик.
- **Кнопки:** «Готово», «Пропустить», «Удалить последнее фото», «Отменить».
- **Ошибки:** лимит, неподдерживаемый MIME/документ, превышение размера, видео/архив/исполняемый файл, повтор `unique_file_id`, фото после завершения этапа.

## Дата, время и контакты

- **Старт:** завершён этап фото.
- **FSM:** `booking.date → slot → name → phone → consent → final`.
- **Шаги:** показываются будущие даты внутри горизонта с хотя бы одним свободным слотом; неделя листается; после выбора слот удерживается; имя предлагается из Telegram или вводится; телефон — contact button/ручной ввод; пользователь даёт согласие и видит итог.
- **Кнопки:** даты, слоты, «Выбрать другое», «Поделиться номером», «Использовать имя», «Согласен», «Отправить заявку», «Изменить контакты/дату/услуги».
- **Ошибки:** дата вне горизонта, занятый/заблокированный/истёкший hold, имя не 2…80 символов или только цифры, телефон невалиден, согласие отсутствует.
- **Финальная операция:** в одной транзакции повторно проверяется слот, создаются Vehicle/Booking/связи услуг/фото, сохраняется AuditLog; повторное нажатие возвращает ту же запись.

## Подтверждение администратора

- **Старт:** новая pending-заявка в `/admin` или карточке-уведомлении.
- **FSM:** `admin.booking_action`, при произвольной причине — `admin.rejection_reason`.
- **Шаги:** просмотр данных/фото → подтвердить, предложить время либо отклонить → при необходимости причина → отправка сообщения клиенту.
- **Кнопки:** «Подтвердить», «Предложить другое время», «Отклонить», «Написать клиенту», «Позвонить», «Профиль», «История».
- **Ошибки:** неадминистратор, несуществующая/чужая/уже финальная заявка, slot conflict. Повторное подтверждение не создаёт новые уведомления.

## Просмотр, отмена и перенос

- **Старт:** «Моя запись».
- **FSM отмены:** `cancel.confirm → cancel.reason`; **FSM переноса:** `reschedule.date → slot → confirm`.
- **Шаги:** показывается ближайшая active-запись; перенос разрешён pending/confirmed, до визита и вне минимального запретного интервала; новая дата/слот подтверждаются; отмена требует явного подтверждения и причины.
- **Кнопки:** «Перенести», «Отменить», «Оставить запись», причины отмены, даты/слоты, «Подтвердить перенос».
- **Ошибки:** нет активной записи, действие после визита, конечный статус, срок переноса, недоступный новый слот. Перенос атомарно освобождает старый слот, бронирует новый, отменяет старые и создаёт новые reminders.

## Каталог, FAQ и расчёт

- **Каталог:** «Услуги и цены» → категории → услуга; показывает состав, длительность, цену от и кнопки «Записаться»/«Рассчитать». Данные только из БД; страницы для длинных списков.
- **FAQ:** «Частые вопросы» → категория/поиск → вопрос → ответ; кнопки «Назад», «Не нашли ответ? Менеджер»; пустой поиск не является ошибкой.
- **Калькулятор:** FSM `calculator.vehicle → service → class → condition → options → result`; выбираются услуга, класс/размер, состояние и опции; результат — диапазон и дисклеймер. Отсутствующее правило предлагает осмотр, не выдаёт вымышленную цену.

## Связь с менеджером

- **Старт:** «Связаться с менеджером».
- **FSM:** `manager.message → photos → phone → preview`.
- **Шаги:** текст 3…2000 символов, до 3 фото, существующий или новый телефон, preview, отправка `ManagerRequest` и карточки менеджеру.
- **Кнопки:** «Готово», «Пропустить», «Изменить», «Отправить», «Отменить».
- **Ошибки:** превышение фото, пустое сообщение, невалидный телефон; при сбое доставки обращение остаётся в БД как open и ставится на повтор.
